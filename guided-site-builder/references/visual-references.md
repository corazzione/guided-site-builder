# Visual References

Visual references are first-class inputs. The user can build from their own idea or attach screenshots, images, links, files, folders, logos, palettes, or competitor examples.

## When To Ask

Ask about references at:

- Stage 1 for initial taste and anti-references;
- Stage 4 for design system;
- Stage 5 for visual identity;
- Stage 7 for navbar/footer;
- Stage 8 for hero;
- Stage 9 for homepage blocks;
- Stage 15 for final visual review.

## What To Extract

Do not copy blindly. Extract:

- structure;
- hierarchy;
- spacing;
- density;
- palette;
- typography;
- border style;
- button style;
- media treatment;
- menu behavior;
- brand feeling;
- motion cues;
- mobile implications;
- what the user wants to avoid.

## Questions

Use these prompts:

```text
What do you like about this reference?
What should I avoid copying?
Is this reference for the whole site or only one section?
Should the final result feel similar, loosely inspired, or only use one detail?
Do you want me to decide how to apply it?
```
