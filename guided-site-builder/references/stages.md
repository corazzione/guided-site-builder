# Detailed Stages

## Stage 1: Niche And Context

Goal: understand the website before any visual decision.

Ask:

- What niche is this website for?
- Is it for a company, solo professional, product, event, SaaS, store, restaurant, clinic, law firm, portfolio, or something else?
- Is this a new website or a redesign?
- What is the main goal: sell, capture leads, receive quote requests, book appointments, present authority, show a portfolio, educate, sell products, build a waitlist, or something else?
- Who is the target audience?
- Do you have competitors or references?
- Are there styles you dislike?
- Are there deadlines, budget limits, tech constraints, or content constraints?

Always offer:

```text
If you do not know, I can decide based on the niche and best practices.
```

## Stage 2: Market Research

Goal: avoid building in a vacuum.

Research or ask about:

- direct competitors;
- common site structures in the niche;
- expected CTAs;
- visual patterns;
- differentiation opportunities;
- common user objections;
- trust signals;
- conversion funnel.

## Stage 3: Architecture And Flowchart

Goal: decide pages, navigation, CTA destinations, and user flow.

Ask:

- How many pages should the site have?
- Should it be simple or complete?
- Will it have a blog?
- Will each service need its own page?
- Will it have forms?
- Where does the main CTA lead: WhatsApp, form, checkout, calendar, call, email, quote page, or something else?
- What path should a visitor follow before converting?

Output a Mermaid flowchart.

## Stage 4: Auxiliary Skills And Design System

Goal: choose the design workflow and define visual rules.

Recommend design skills when available:

- `impeccable` for UI/UX, design systems, hierarchy, responsiveness, polish;
- `ui-ux-pro-max` for visual direction, palettes, components, layouts;
- `frontend-design` for real interface generation;
- `using-superpowers` or equivalent discovery workflows when available.

Ask:

- Do you already have a logo?
- Do you have a palette?
- Do you have fonts?
- Do you have a brand manual?
- Should I create a provisional identity?
- Should the site feel minimal, premium, vibrant, institutional, modern, editorial, technical, handcrafted, luxurious, or direct?

## Stage 5: Visual Identity

Goal: remove visual ambiguity before building blocks.

Collect:

- logo;
- color palette;
- fonts;
- images;
- icons;
- tone;
- visual references;
- anti-references;
- boldness level;
- photography/illustration style;
- density;
- borders;
- shadows;
- texture;

## Stages 6-15

For later stages, follow the same pattern:

1. Ask only the questions needed for the current stage.
2. Offer variants or decisions.
3. Show real previews when visual.
4. Refine the chosen direction.
5. Produce `Copy choices`.

Stage order:

- Stage 6: Assets and base content.
- Stage 7: Navbar and footer.
- Stage 8: Hero section.
- Stage 9: Homepage blocks.
- Stage 10: Internal pages.
- Stage 11: Forms and conversion.
- Stage 12: Responsiveness and states.
- Stage 13: Animations and microinteractions.
- Stage 14: SEO, performance, accessibility.
- Stage 15: Final review and delivery.
