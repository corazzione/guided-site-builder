# Use Cases

Use the skill for complete websites and for focused UI decisions.

## Full Website

User request:

```text
Use /site-start to build a website for my clinic.
```

Behavior:

- run all stages;
- ask about references;
- build architecture;
- create visual labs;
- produce copyable decisions;
- implement after approval.

## Architecture Only

User request:

```text
Use /site-architecture for a restaurant website.
```

Behavior:

- ask business and CTA questions;
- recommend pages;
- create a flowchart;
- produce copyable architecture summary.

## Navbar Only

User request:

```text
Use /site-navbar. I want 10 navbar variants for a premium beauty clinic.
```

Behavior:

- ask for logo, menu, CTA, references, and design system;
- generate a lab with variants;
- include desktop/mobile and dropdown behavior;
- refine selected base;
- produce copy choices.

## Toasts Only

User request:

```text
Use /component-lab to generate toast notifications for a finance dashboard.
```

Behavior:

- ask status types: success, error, warning, info, loading;
- ask placement and duration;
- ask tone: calm, urgent, premium, playful;
- generate variants;
- refine animation, icon, color, size, and stacking;
- produce copy choices.

## Pop-Ups Only

User request:

```text
Use /component-lab for pop-ups on a SaaS landing page.
```

Behavior:

- ask purpose: lead capture, discount, onboarding, exit intent, update, warning;
- ask trigger: time, scroll, button, exit intent;
- ask dismissal and accessibility rules;
- generate variants;
- refine copy, layout, motion, backdrop, and mobile behavior;
- produce copy choices.

## Hero Only

User request:

```text
Use /site-hero. I have a screenshot reference.
```

Behavior:

- analyze the reference;
- ask what to borrow and avoid;
- generate hero variants;
- refine headline, media, CTA, proof, and mobile layout;
- produce copy choices.

## Existing Project Integration

User request:

```text
Use /component-lab to create testimonial variants that fit my existing site.
```

Behavior:

- inspect existing project styles when available;
- ask for missing design context;
- preserve existing design system;
- create variants that can fit into the project;
- produce copy choices and implementation plan.
